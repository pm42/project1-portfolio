Object Oriented Programming

Introduction

Object-Oriented Programming or OOPs refers to languages that uses objects in programming. Object-oriented programming aims to implement real-world entities like inheritance, abstraction, polymorphism and encapsulation in programming. The main aim of OOP is to bind together the data and the functions that operate on them so that no other part of the code can access this data except that function.

This article explains the above principles of object-oriented programming using the calculator program developed by me.

Encapsulation

Encapsulation is one of the fundamental concepts in object-oriented programming (OOP). It describes the idea of wrapping data and the methods that work on data within one unit. This puts restrictions on accessing variables and methods directly and can prevent the accidental modification of data.

Encapsulation in Calculator

A class is an example of encapsulation as it encapsulates all the data that is member functions, variables, etc.

Encapsulation is achieved when each object keeps its state private, inside a class. Other objects don’t have direct access to this state. Instead, they can only call a list of public functions — called methods.

As we can see in the above figure, Calculation class encapsulates all the data and methods. Each instance of this class keep the state of variable values as private. values property is not accessible directly from outside Calculation class. Only way to access this variable is using the method get_values().

In this way encapsulation is achieved in Calculator

Abstraction

Abstraction means that each object should only expose a high-level mechanism for using it and this mechanism should hide internal implementation details. It should only reveal operations relevant for the other objects.

Abstraction in Calculator

As we can see in the above figure (right) get_result method uses the divide method of Division class to get the division result. But the get_result method has no idea on how the calculation is being performed and from its perspective it only cares about the division result and it is least bothered the underlying functionality.

Division class hides its implementation(abstraction) from other classes and exposes a high level mechanism via divide method to use it.

In this way abstraction is achieved in Calculator.

Inheritance

Inheritance is the capability of one class to derive or inherit the properties from another class.The benefits of inheritance are:

  • It represents real-world relationships well
  • It provides the re-usability of a code
  • It is transitive in nature

Inheritance in Calculator

Here Multiplication is the derived class and Calculation is the base class. As derived class inherits the properties of base class, Multiplication derived class is able to access get_values() method though it is defined in the base class.

Division class hides its implementation(abstraction) from other classes and exposes a high level mechanism via divide method to use it.

In this way inheritance is achieved in Calculator and allow us to reuse the code from Calculation class.

Polymorphism

In programming, polymorphism means the same function name (but different signatures) being used for different types.

Using polymorphism it is possible to modify a method in a child class that it has inherited from the parent class. This is particularly useful to modify the behavior of the parent method based on the child class need.

Polymorphism in Calculator

Addition and Multiplication classes are derived from the parent class Calculation. get_result() method is an abstract method in parent class (no method implementation).

Both Addition and Multiplication classes implemented its own behavior for get_result() method thereby achieving polymorphism.
