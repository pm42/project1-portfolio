AAA Testing

Software Testing

Software testing is an essential part of the software development process and is done so that only the best product reaches the end-user. It is used to test programs and applications to see if they function according to the specifications.

Importance

If any problems persist, they should ideally be taken care of earlier on so that they don’t show up when the final version is sent out to clients. Software unit testing mainly involves checking individual modules for errors and helps to fix errors at the earlier stage itself.

AAA - A Pattern for writing good tests

  • Arrange set up the inputs and targets
  • Act cover the main thing to be tested. This could be calling a function or method or interacting with a web page
  • Assert expected outcomes. This step determines if the test passes or fails

Pytest

pytest is a mature full-featured Python testing tool that helps us write better programs.

It is a framework that makes it easy to write small tests, yet scales to support complex functional testing for applications and libraries.

Features of Pytest

  • Detailed info on failing assert
  • Auto-discovery of test modules
  • Modular fixtures for managing resources
  • Rich plugin architecture

AAA - Unit Testing

Here AAA pattern is used to perform unit testing for the division module.

Division by zero is not possible, so we are using AAA pattern to assert that when zero is passed for division, our module throws appropriate error instead of performing the division operations.

  • Arrange step initializes the input data as tuples
  • Act step then calls the divide static method in calculator class and passes the input data created in arrange step as argument and stores the result in variable result
  • Assert step then verifies if the variable result contains the error message cannot divide by zero instead of any numeric values

AAA - Feature Testing

Here AAA pattern is used to perform feature testing i.e. Used to check whether the index page is loading properly and the menu link in the index page is working or not.

  • Arrange step specifies the URL(in this case index page url) we are going to check
  • Act step then calls the API get using “requests” with URL as the argument and then parse the response’s body from JSON into a Python dictionary and stores it in response object
  • Assert step then verifies that the HTTP status code is 200, meaning “OK” or “success,” and that the word “Home” appears somewhere in the response’s abstract text.
