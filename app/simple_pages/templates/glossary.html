Glossary - OOPs terms with definition

Object Oriented Programming

Object Oriented programming (OOP) is a programming paradigm that relies on the concept of classes and objects. It is used to structure a software program into simple, reusable pieces of code blueprints (usually called classes), which are used to create individual instances of objects.

Class

A class is a collection of objects. A class contains the blueprints or the prototype from which the objects are being created. It is a logical entity that contains some attributes and methods.

Objects

object is an entity that has a state and behavior associated with it. It consists of State, Behavior and Identity.

NameSpace

A namespace is a collection of currently defined symbolic names along with information about the object that each name references. Python consists of four types of namespaces:

  • Built-In
  • Global
  • Enclosing
  • Local

Constructors

Constructors are generally used for instantiating an object. The task of constructors is to initialize(assign values) to the data members of the class when an object of the class is created.

In Python the __init__() method is called the constructor and is always called when an object is created.

Constructor Types
Default Constructor

Simple Constructor which doesn't accept any arguments

Type Hint

Type hint means providing hints to function to only accept the given data type.It is a method by which we can force function to accept the desired data type.

Type Cast

Converting one datatype into another is known as type casting. It can be either Implicit or Explicit type casting.

Unit Test

Unit testing involves testing individual components of the software program or application. The main purpose behind this is to check that all the individual parts are working as intended.

A unit is known as the smallest possible component of software that can be tested. Generally, it has a few inputs and a single output.

Advantages of Unit Testing
  • Process becomes agile
  • Quality of code improves/li>
  • Facilitates Change
  • Reduce Costs
  • Improves Design
  • Find Software Bugs Easily

Fixture

In testing, a fixture provides a defined, reliable and consistent context for the tests. This could include environment (for example a database configured with known parameters) or content (such as a dataset).

Fixtures define the steps and data that constitute the arrange phase of a test. Fixtures are designed to be explicit, modular and scalable.

Unit Testing Process

  • Create the test cases
  • Review or Rework
  • Baseline
  • Execute TestCase

Property

Property are characteristics or attributes of an object

Method

Methods are functions defined inside the body of a class. They are used to define the behaviors of an object.

Static Property

A static property of a class is shared by all instances of that class. Static keyword is used to define a static property

Static Method

A static method is a method that is bound to the class and not the object of the class. A static method can’t access or modify the class state. Static methods are used to create utility functions.

Instance Property

Instance properties are those properties that are defined inside any class and require an instance to access them. Without creating instances we cannot access these properties

Instance Method

An instance method is a method that belongs to the instances of a class, not to the class itself.An instance method can access and modify the value of attributes of an instance.

Encapsulation

It is the idea of wrapping data and the methods that work on data within one unit. This puts restrictions on accessing variables directly and a variable can only be changed by an object’s method. A class is an example of encapsulation

Abstraction

Abstraction means that each object should only expose a high-level mechanism for using it. This mechanism should hide internal implementation details. It should only reveal operations relevant for the other objects.

Inheritance

Inheritance is the capability of one class to derive or inherit the properties from another class. The class that derives properties is called the derived class or child class and the class from which the properties are being derived is called the base class or parent class.

Polymorphism

Polymorphism lets us define methods in the child class that have the same name as the methods in the parent class.Polymorphism means the same function name (but different signatures) being used for different types.

SOLID - An Acronym for the fist five object-oriented principles by Robert C.Martin

  • S-Single-responsibility Principle

    A class should have one and only one reason to change, meaning that a class should have only one job.

  • O-Open-closed Principle

    Objects or entities should be open for extension but closed for modification.

  • L-Liskov Substitution Principle

    Every subclass or derived class should be substitutable for their base or parent class

  • I-Interface Segregation Principle

    A class should never be forced to implement an interface that it does not use, or class shouldn’t be forced to depend on methods they do not use.

  • D-Dependency Inversion Principle

    Entities must depend on abstractions, not on concretions. High-level module must not depend on the low-level module, but they should depend on abstractions.

Pylint

Pylint is a tool that checks for errors in Python code, tries to enforce a coding standard and looks for code smells. The default coding style used by Pylint is close to PEP 8.

Pylint will display a number of messages as it analyzes the code and looks for type errors and recommend suggestions about how particular blocks can be refactored.

PEP8

A Style Guide for Python Code. It gives coding conventions for the Python code comprising the standard library in the main Python distribution.

Design Patterns : Pre-made blueprints that we can customize to solve a recurring design problem in our code
Sections of a Pattern
  • Intent describes both the problem and the solution
  • Motivation explains the problem and the solution the pattern makes possible
  • Structure shows each part of the pattern and how they are related
Classification of Patterns
  • Creational patterns provide object creation mechanisms that increase flexibility and reuse of existing code
  • Structural patterns explain how to assemble objects and classes into larger structures, while keeping these structures flexible and efficient
  • Behavioral patterns take care of effective communication and the assignment of responsibilities between objects
